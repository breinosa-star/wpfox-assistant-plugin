YOUR TASK — build this page by calling tools in this order:

1. Call sb_query_templates. Study the template outlines to understand how the theme designer structured pages of this type. Templates are guidelines only — you will NOT assign a template to the page. Use the outlines to inform which patterns best suit this page's structure.

2. Call sb_query_patterns. Each pattern in the response includes a content_schema — a flat indexed list of every fillable slot in that pattern. Read each slot carefully: it has an index, a type, and a hint showing the current placeholder text. You will use these indices when calling sb_append_pattern.

3. Call sb_create_page with the page title. Note the post_id returned — you must pass it to every subsequent call.

4. For each selected pattern, call sb_append_pattern. The slots parameter is REQUIRED — it must contain one entry per slot in that pattern's content_schema. Build the slots array directly from the content_schema indices you received in step 2:

   - Heading slot:   {"index": <n>, "content": "<short punchy headline from KB>"}
   - Paragraph slot: {"index": <n>, "content": "<2-3 sentences from KB>"}
   - Buttons slot:   {"index": <n>, "buttons": [{"index": 0, "text": "<label>"}, {"index": 1, "text": "<label>"}]}
   - Image slot:     {"index": <n>, "keyword": "<2-4 word stock photo search term>"}
   - HTML slot:      {"index": <n>, "html": "<rewritten HTML with KB content, all classes/ids preserved>"}

   After each sb_append_pattern call, read the post_content returned in the response. Verify that your content was actually injected — you should see your text in the block HTML. If slots_filled is 0 or your content is missing, call sb_remove_last_pattern to undo the bad append, then retry sb_append_pattern with a corrected slots array. Do not call sb_page_complete until all patterns show real content in post_content.

5. Call sb_page_complete only after verifying post_content looks correct.

- Hero patterns (full-width, large background image, prominent headline and CTA) are ONLY appropriate for the home page or landing pages (page_type: "home" or "landing").
- For all other page types do NOT use a hero as the intro — choose a simpler opening pattern or start directly with the first relevant content pattern.
- Vary the lead pattern across pages. Do not open every page with the same pattern.

PATTERN SELECTION RULES:

- Order them logically: intro first, supporting content in the middle, call-to-action last.
- If sb_query_patterns returns no patterns, call sb_page_complete immediately with a note explaining no patterns were available.
